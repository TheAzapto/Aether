Modular Design

Engine Architecture

Seven focused modules, each handling a single responsibility — from linear algebra primitives to GPU-accelerated rendering.

📐

Vector3

Full 3D vector with add, subtract, scalar multiply, dot product, cross product, normalization, and epsilon-aware comparison.

dot() cross() normalized() length
🧮

Matrix4

4×4 column-major matrix with perspective & orthographic projections, look-at view, translation, rotation, and scale transforms.

perspective() look_at() rotation_xyz()
🧊

RigidBody

State container for position, velocity, inverse mass, and collision shape. Supports both dynamic bodies and static anchors (mass = 0).

position velocity inv_mass restitution
🌍

World

The simulation controller — manages bodies, applies force generators, runs fixed-timestep sub-stepping at 2ms intervals, and resolves collisions each step.

step() add_force() sub-stepping
💥

Collision

Dispatches shape-pair collision detection: Circle↔Circle, Circle↔AABB, AABB↔AABB, and SAT-based Polygon detection with contact normals & penetration depth.

SAT Minkowski CollisionResult
🎨

Renderer & Camera

Modern OpenGL 3.3+ core-profile renderer with GLSL shaders, VBOs, and a fixed-position camera system with view and projection matrices.

GLSL 330 VBO MVP
Capabilities

Engine Features

Everything you need for a physics simulation, built from the ground up with no external physics libraries.

🔺

4 Shape Primitives

Circle, AABB (rectangle), convex Polygon, and Line — each with area calculation, bounding radius, and point containment checks.

⚡

SAT Collision Detection

Separating Axis Theorem for convex polygon pairs, Minkowski-sum approach for Circle↔AABB, and optimized circle-to-circle distance checks.

⏱️

Fixed-Timestep Sub-Stepping

Internal physics runs at 2ms steps regardless of frame rate, preventing tunneling artifacts and ensuring deterministic simulation.

🌊

Force Generators

Pluggable force system with Gravity (F = mg), Drag (-kv), and Friction (μN opposing motion) — easily extend with custom generators.

🖥️

GPU Rendering (OpenGL 3.3+)

Modern core-profile pipeline with vertex/fragment shaders, dynamic VBOs, and per-body MVP matrix transforms — all vertex processing on the GPU.

📷

3D Camera System

Perspective camera with configurable FOV, aspect ratio, near/far planes, and cached view-projection matrices for efficient rendering.

Code

Intuitive API

Set up a full physics simulation in just a few lines. Create bodies, add forces, step the world — Aether handles the math, collisions, and rendering for you. 

# Create the physics world
from main import World, RigidBody, Gravity
from Vector import Vector3
from Primitives import Circle

# World at ~60 FPS delta
world = World(delta=0.016)

# Add a bouncing ball
ball = RigidBody(
    p=Vector3(250, 300),
    v=Vector3(50, 0),
    shape=Circle(10)
)
world.add(ball)
world.add_force(Gravity(-9.81))

# Run the simulation loop
while True:
    world.step()
# Different body types
from Primitives import Circle, AABB, Polygon

# Dynamic circle (mass = 1.0)
ball = RigidBody(
    p=Vector3(100, 200),
    shape=Circle(radius=5)
)

# Static platform (mass = 0)
platform = RigidBody(
    p=Vector3(250, 50),
    m=0,
    shape=AABB(100, 10)
)

# Regular hexagon
hex = Polygon.create_regular(
    sides=6,
    radius=15
)
# Built-in force generators
from main import Gravity, Drag, Friction

# Pull bodies downward: F = mg
gravity = Gravity(g=-9.81)

# Air resistance: F = -kv
drag = Drag(k=0.05)

# Surface friction: F = -μN
friction = Friction(
    mu=0.2,
    gravity=9.81
)

# Add all at once
world.add_force([gravity, drag, friction])
Quick Start

Get Started

Three steps to start simulating physics with Aether.

1

Clone the Repository

Grab the source code from GitHub and enter the project directory.

$ git clone https://github.com/TheAzapto/Aether.git
$ cd Aether
2

Install Dependencies

Set up a virtual environment and install the required Python packages.

$ python -m venv .venv
$ pip install -r requirements.txt
3

Run the Simulation

Launch the demo and watch 50 bodies interact with gravity, drag, and collisions.

$ python app.py
